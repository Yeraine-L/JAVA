# 校园流浪猫狗投喂与领养管理系统 - 测试与验收文档

## 1. 测试目标

验证系统核心业务流程、角色权限、异常处理、数据库读写、界面交互均符合需求文档，确保项目可以稳定运行并用于答辩演示。

## 2. 测试范围

- 登录与注册
- 动物展示与检索
- 领养申请与审核
- 用户中心
- 公告与消息通知
- 投喂管理
- 异常上报
- 物资管理
- 后台管理
- 图表统计
- 系统日志

## 3. 测试环境

- 操作系统：Windows
- JDK：8 及以上
- 数据库：MySQL 8.0
- 启动方式：IDEA 运行 Main 类

## 4. 核心验收流程

### 流程一：注册与登录

- 普通用户成功注册
- 志愿者成功注册
- 管理员使用固定账号登录
- 注册时校验用户名重复
- 注册时校验手机号格式
- 登录失败时提示账号或密码错误

验收标准：

- 所有角色能正确进入对应界面
- 错误提示明确
- 注册数据正确写入数据库

### 流程二：领养闭环

- 用户查看动物列表
- 用户查看动物详情
- 用户提交领养申请
- 管理员审核通过或拒绝
- 用户在个人中心查看结果
- 审核通过后生成协议与消息通知

验收标准：

- 申请状态流转正确
- 已领养动物状态正确更新
- 审核意见可查看
- 消息提醒可查看

### 流程三：投喂管理闭环

- 志愿者提交投喂点申请
- 管理员审核投喂点
- 志愿者新增投喂记录
- 管理员查看投喂记录
- 志愿者异常上报
- 管理员处理异常

验收标准：

- 投喂点状态变化正确
- 投喂记录可查询
- 异常状态可更新
- 志愿者可查看处理结果

### 流程四：物资管理闭环

- 管理员新增物资
- 管理员或志愿者执行入库
- 执行出库并扣减库存
- 库存不足时阻止出库
- 查看出入库记录

验收标准：

- 库存变化准确
- 记录可追溯
- 低库存有明显提示

## 6. 异常测试

- 数据库连接失败
- 图片文件不存在
- 输入框为空
- 手机号非法
- 数值输入非数字
- 重复提交
- 删除不存在的数据

验收标准：

- 不闪退
- 有明确提示
- 不写入错误数据

## 7. 性能与稳定性验收

- 普通查询响应时间控制在 1 秒内
- 表格加载不出现明显卡顿
- 弹窗打开关闭稳定
- 多次切换页面不出现空白或报错

## 8. 答辩前检查清单

- 所有测试账号可登录
- 数据库连接正常
- 测试数据已导入
- 关键流程已走通
- 默认图片存在
- 公告、消息、物资、动物数据都有演示样例
- 管理员审核功能正常
- 图表页面可正常显示

- 数据库脚本完整

- 关键异常已处理
- 有测试数据
