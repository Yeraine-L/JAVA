# 校园流浪猫狗投喂与领养管理系统 - 系统设计说明书

## 1. 系统定位

本系统是一个基于 Java Swing 的本地桌面应用，用于校园流浪猫狗的投喂与领养管理，支持普通用户、志愿者、管理员三类角色，强调本地可运行、功能完整、流程闭环、便于演示。

## 2. 技术架构

- 开发语言：Java
- JDK 版本：JDK 8 及以上
- 界面技术：Swing
- 数据存储：MySQL 8.0
- 数据访问：JDBC
- 图表工具：JFreeChart
- 图片处理：Java ImageIcon、BufferedImage

## 3. 系统分层设计

### 3.1 分层结构

建议采用以下分层：

- view：界面层，负责窗口、面板、表单、表格、弹窗
- controller：事件协调层，负责按钮事件和页面跳转
- service：业务层，负责业务规则、状态流转、权限校验
- dao：数据访问层，负责数据库增删改查
- entity：实体层，负责系统核心对象
- utils：工具层，负责校验、弹窗、图片、时间、数据库连接等通用能力
- db：数据库脚本或数据库初始化相关内容
- config：系统配置类与常量

### 3.2 推荐包结构

```text
src
 └─ com.campuspet
    ├─ Main.java
    ├─ config
    ├─ controller
    ├─ dao
    ├─ db
    ├─ entity
    ├─ enums
    ├─ service
    ├─ utils
    └─ view
       ├─ common
       ├─ login
       ├─ user
       ├─ volunteer
       └─ admin
```

## 4. 角色与权限设计

### 4.1 普通用户

可操作功能：

- 注册与登录
- 查看动物列表与详情
- 搜索筛选动物
- 提交领养申请
- 查看申请结果
- 查看个人消息
- 修改个人信息
- 查看领养回访记录

### 4.2 志愿者

具备普通用户的浏览能力，并额外支持：

- 申请投喂点
- 记录投喂日志
- 上报异常情况
- 查看自己的投喂点
- 查看自己的投喂记录
- 参与物资相关操作

### 4.3 管理员

具备最高权限，支持：

- 管理动物档案
- 审核领养申请
- 审核投喂点申请
- 处理异常上报
- 管理物资
- 发布公告
- 管理用户
- 查看统计图表
- 查看系统日志

## 5. 功能模块设计

### 5.1 登录注册模块

- LoginFrame：登录窗口
- RegisterDialog：注册弹窗
- LoginService：登录、注册、校验逻辑

### 5.2 主界面模块

- MainFrame：系统主窗口
- SideMenuPanel：左侧菜单
- ContentPanelContainer：右侧内容区域

### 5.3 动物管理模块

- PetListPanel：动物列表
- PetDetailDialog：动物详情
- PetSearchPanel：搜索筛选区域
- CampusMapPanel：校园静态地图标记

### 5.4 领养模块

- AdoptApplyDialog：领养申请弹窗
- AdoptApplyManagePanel：管理员审核界面
- AdoptAgreementDialog：协议弹窗
- FollowUpPanel：领养回访记录

### 5.5 投喂管理模块

- FeedPointApplyDialog：投喂点申请
- FeedPointManagePanel：投喂点审核与管理
- FeedRecordPanel：投喂记录列表与录入
- ExceptionReportDialog：异常上报

### 5.6 物资管理模块

- GoodsPanel：库存列表
- GoodsInDialog：入库弹窗
- GoodsOutDialog：出库弹窗
- GoodsRecordPanel：出入库记录

### 5.7 公告与通知模块

- AnnouncementPanel：公告列表与管理
- AnnouncementDialog：公告新增编辑弹窗
- MessageCenterPanel：消息中心

### 5.8 统计与日志模块

- StatisticsPanel：图表展示页
- LogPanel：系统日志页

## 6. 页面跳转设计

### 6.1 登录后跳转

- 普通用户：进入用户主界面
- 志愿者：进入志愿者主界面
- 管理员：进入管理员主界面

### 6.2 主界面导航

主窗口采用左侧导航栏 + 右侧内容区结构：

- 左侧点击菜单
- 右侧切换对应 JPanel
- 弹窗操作统一使用 JDialog

## 7. 状态流转设计

### 7.1 领养申请状态

- 待审核
- 已通过
- 已拒绝

规则：

- 审核通过后，宠物状态更新为已领养
- 自动生成领养协议
- 自动向申请人写入消息通知

### 7.2 投喂点状态

- 待审核
- 已生效
- 已禁用

### 7.3 异常上报状态

- 待处理
- 已处理

### 7.4 动物状态

- 待领养
- 已领养
- 救助中

## 8. 核心业务规则

- 用户名唯一
- 手机号必须为 11 位数字
- 密码长度不少于 6 位
- 用户不能重复申请同一只动物
- 已领养动物不能再申请
- 审核意见为必填
- 物资出库前必须检查库存
- 删除操作需要二次确认
- 图片加载失败显示默认占位图

## 9. 异常处理设计

- 所有数据库操作使用 try-catch
- 所有表单提交先做必填校验
- 所有图片显示支持失败兜底
- 所有删除类操作统一二次确认
- 所有关键失败都弹窗提示用户
- 所有关键操作写入系统日志

## 10. UI 统一规范

- 主色调保持统一
- 表格统一使用 JTable
- 弹窗统一使用确认和取消按钮
- 按钮尺寸、字体、边距统一
- 所有提示统一使用封装的消息弹窗工具
- 页面尽量使用滚动面板，避免内容溢出

## 11. 初版交付边界

首版优先完成以下闭环：

- 注册
- 登录
- 动物列表查看
- 领养申请

- 用户查看审核结果
- 公告与消息通知基础能力

首版完成后，再逐步补充：

- 投喂管理
- 物资管理
- 图表统计
