# 异常上报模块开发文档

## 1. 页面与弹窗清单

### 异常上报面板（志愿者）
- **文件**：`src/main/java/com/campuspet/view/volunteer/ExceptionReportPanel.java`
- **功能**：志愿者上报异常情况
- **布局**：表单输入 + 图片上传 + 提交按钮

### 异常列表面板（管理员）
- **文件**：`src/main/java/com/campuspet/view/admin/ExceptionListPanel.java`
- **功能**：管理员查看所有异常上报
- **布局**：表格展示 + 处理按钮

### 异常处理弹窗（管理员）
- **文件**：`src/main/java/com/campuspet/view/admin/HandleExceptionDialog.java`
- **功能**：管理员处理异常上报
- **布局**：异常详情 + 处理意见 + 处理按钮

### 异常详情弹窗（用户）
- **文件**：`src/main/java/com/campuspet/view/user/ExceptionDetailDialog.java`
- **功能**：查看异常详情
- **布局**：异常信息 + 处理状态 + 处理意见

## 2. 数据库表与字段

### exception_report 表（新建）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| volunteer_id | BIGINT | 志愿者ID |
| volunteer_name | VARCHAR(50) | 志愿者姓名 |
| location | VARCHAR(100) | 异常地点 |
| area | VARCHAR(100) | 所在区域 |
| type | VARCHAR(50) | 异常类型（受伤/生病/危险/其他） |
| description | TEXT | 异常描述 |
| photo | VARCHAR(255) | 图片路径 |
| status | VARCHAR(20) | 状态（待处理/处理中/已处理） |
| handle_opinion | TEXT | 处理意见 |
| handle_time | DATETIME | 处理时间 |
| create_time | DATETIME | 创建时间 |

## 6. 界面层开发

### ExceptionReportPanel.java（志愿者）

**功能**：
- 选择异常类型（下拉框）
- 输入异常地点
- 输入所在区域
- 输入异常描述
- 上传图片（可选）
- 提交上报

**校验**：
- 异常类型不能为空
- 异常地点不能为空
- 异常描述不能为空

### ExceptionListPanel.java（管理员）

**功能**：
- 表格展示所有异常上报
- 显示：志愿者、地点、类型、状态、创建时间
- 处理按钮 → 打开 HandleExceptionDialog

### HandleExceptionDialog.java（管理员）

**功能**：
- 显示异常详情（志愿者、地点、类型、描述、图片）
- 选择处理状态（待处理/处理中/已处理）
- 输入处理意见
- 提交处理

**校验**：
- 处理意见不能为空

### ExceptionDetailDialog.java（用户）

**功能**：
- 显示异常详情
- 显示处理状态
- 显示处理意见

## 7. 绑定事件逻辑

### ExceptionReportPanel.java（志愿者）
- 提交按钮 → 调用 ExceptionReportService.report()

### ExceptionListPanel.java（管理员）
- 处理按钮 → 打开 HandleExceptionDialog（传入异常ID）
- 刷新按钮 → 重新加载异常列表

### HandleExceptionDialog.java（管理员）
- 提交按钮 → 调用 ExceptionReportService.handle(id, status, opinion)
- 取消按钮 → 关闭弹窗

## 8. 表单校验

### ExceptionReportPanel.java
- 异常类型不能为空
- 异常地点不能为空
- 异常描述不能为空

### HandleExceptionDialog.java
- 处理意见不能为空

## 10. 正常流程测试

1. 志愿者登录后进入异常上报
2. 选择异常类型，填写异常信息，点击提交
3. 异常上报成功
4. 管理员登录后进入异常管理
5. 查看异常列表
6. 点击"处理"按钮
7. 填写处理意见，点击提交
8. 异常处理成功
9. 志愿者登录后查看异常详情
10. 异常状态和处理意见正确显示

## 11. 异常流程测试

1. 上报异常时类型为空 → 提示"异常类型不能为空"
2. 上报异常时地点为空 → 提示"异常地点不能为空"
3. 上报异常时描述为空 → 提示"异常描述不能为空"
4. 处理异常时意见为空 → 提示"处理意见不能为空"
5. 处理不存在的异常 → 提示"异常不存在"
6. 数据库操作失败 → 提示"操作失败，请重试"
