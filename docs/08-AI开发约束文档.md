# 校园流浪猫狗投喂与领养管理系统 - AI 开发约束文档

## 1. 文档目的

本文件用于约束 Trae 在全程 AI 开发过程中的输出风格、工程结构、编码习惯和实现边界，减少返工和代码风格不一致问题。

## 2. 总体约束

- 项目必须使用纯 Java 开发
- GUI 必须使用 Swing
- 数据存储使用 MySQL + JDBC
- 所有功能必须对应需求文档中的功能点
- 优先保证功能闭环，不优先追求复杂美化
- 不引入与项目无关的第三方框架

## 3. 工程结构约束

统一使用以下包结构：

- com.campuspet.config
- com.campuspet.controller
- com.campuspet.dao
- com.campuspet.db
- com.campuspet.entity
- com.campuspet.enums
- com.campuspet.service
- com.campuspet.utils
- com.campuspet.view

约束规则：

- entity 只放实体类
- dao 只处理数据库访问
- service 只处理业务逻辑
- view 只处理界面显示和简单事件绑定
- utils 只放通用工具能力

## 4. 命名规范

- 类名使用大驼峰
- 方法名、变量名使用小驼峰
- 常量使用全大写下划线
- 数据表使用小写下划线
- 页面类统一以 Frame、Panel、Dialog 结尾
- DAO 类统一以 Dao 结尾
- Service 类统一以 Service 结尾

## 5. 编码规范

- 所有输入先校验再提交
- 所有数据库操作统一捕获异常
- 所有状态变化走 service 层
- 所有弹窗提示统一封装
- 所有时间获取统一通过工具类
- 所有图片加载统一通过工具类
- 所有删除操作必须二次确认

## 6. 页面开发规范

- 主窗口使用 BorderLayout
- 左侧导航与右侧内容分离
- 内容页尽量做成独立 JPanel
- 新增、编辑、审核统一使用 JDialog
- 表格统一封装表格模型与刷新方法
- 页面初始化时不直接写死测试数据

## 7. 数据访问规范

- JDBC 连接统一由 DbUtil 管理
- 关闭连接、语句、结果集必须封装
- SQL 语句集中管理在 DAO 中
- 不在界面层直接写 SQL
- 涉及状态联动的更新放在事务逻辑中处理

## 8. 业务规则约束

- 用户名唯一
- 手机号必须合法
- 密码长度不少于 6 位
- 同一用户不能重复申请同一只动物
- 已领养动物不能提交申请
- 审核意见不能为空
- 库存不足不能出库
- 管理员固定账号无需注册

## 9. UI 风格约束

- 全局字体、按钮尺寸、颜色尽量统一
- 错误提示、成功提示、警告提示风格统一
- 图片缺失时显示默认占位图
- 表格列名使用中文，便于演示
- 页面操作尽量少层级，减少答辩时点击复杂度

## 10. 测试数据约束

- 必须提供管理员账号
- 必须提供普通用户和志愿者测试账号
- 必须提供多条动物测试数据
- 必须覆盖待领养、已领养、救助中三种状态
- 必须提供公告、物资、投喂点示例数据

## 11. 开发优先级约束

优先级从高到低：

1. 登录与注册
2. 动物展示与搜索
3. 领养申请与审核
4. 用户中心
5. 公告与消息
6. 投喂管理
7. 物资管理
8. 图表统计
9. 系统日志

## 12. 每次开发任务固定执行模式

每次 AI 修改代码时，统一按以下流程推进：

1. 先查看相关文档
2. 再确认页面、实体、数据表
3. 再改代码
4. 再验证主流程
5. 再修复报错
6. 最后更新文档状态

## 13. 禁止事项

- 不跳过 service 层直接在界面里写复杂业务
- 不在多个页面重复写相同校验逻辑
- 不引入不必要的新框架
- 不在未规划的数据表中随意增加字段
- 不让页面命名和数据字段命名前后不一致
