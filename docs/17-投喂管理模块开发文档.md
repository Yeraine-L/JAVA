# 投喂管理模块开发文档

## 1. 页面与弹窗清单

### 投喂点申请面板（志愿者）
- **文件**：`src/main/java/com/campuspet/view/volunteer/FeedPointApplyPanel.java`
- **功能**：志愿者申请投喂点
- **布局**：表单输入 + 提交按钮

### 投喂点列表面板（管理员）
- **文件**：`src/main/java/com/campuspet/view/admin/FeedPointListPanel.java`
- **功能**：管理员审核投喂点申请
- **布局**：表格展示 + 通过/拒绝按钮

### 投喂记录录入面板（志愿者）
- **文件**：`src/main/java/com/campuspet/view/volunteer/FeedRecordInputPanel.java`
- **功能**：志愿者录入投喂记录
- **布局**：表单输入 + 提交按钮

### 投喂记录列表面板（志愿者）
- **文件**：`src/main/java/com/campuspet/view/volunteer/FeedRecordListPanel.java`
- **功能**：志愿者查看自己的投喂记录
- **布局**：表格展示 + 刷新按钮

## 2. 数据库表与字段

### feed_point 表（新建）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| volunteer_id | BIGINT | 志愿者ID |
| volunteer_name | VARCHAR(50) | 志愿者姓名 |
| location | VARCHAR(100) | 投喂地点 |
| area | VARCHAR(100) | 所在区域 |
| description | TEXT | 说明 |
| status | VARCHAR(20) | 状态（待审核/已通过/已拒绝） |
| apply_time | DATETIME | 申请时间 |
| review_time | DATETIME | 审核时间 |
| review_opinion | VARCHAR(255) | 审核意见 |

### feed_record 表（新建）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| feed_point_id | BIGINT | 投喂点ID |
| volunteer_id | BIGINT | 志愿者ID |
| volunteer_name | VARCHAR(50) | 志愿者姓名 |
| pet_count | INT | 宠物数量 |
| food_amount | DECIMAL(10,2) | 食物数量 |
| food_type | VARCHAR(50) | 食物类型 |
| feed_time | DATETIME | 投喂时间 |
| create_time | DATETIME | 创建时间 |

## 6. 界面层开发

### FeedPointApplyPanel.java（志愿者）

**功能**：
- 输入投喂地点
- 输入所在区域
- 输入说明
- 提交申请

**校验**：
- 投喂地点不能为空
- 所在区域不能为空
- 说明不能为空

### FeedPointListPanel.java（管理员）

**功能**：
- 表格展示所有投喂点申请
- 显示：志愿者、地点、区域、申请时间、状态
- 通过按钮 → 审核通过
- 拒绝按钮 → 审核拒绝

### FeedRecordInputPanel.java（志愿者）

**功能**：
- 选择投喂点
- 输入宠物数量
- 输入食物数量
- 选择食物类型
- 选择投喂时间
- 提交记录

**校验**：
- 投喂点不能为空
- 宠物数量必须 ≥ 1
- 食物数量必须 > 0
- 投喂时间不能早于当前时间

### FeedRecordListPanel.java（志愿者）

**功能**：
- 表格展示自己的投喂记录
- 显示：投喂点、宠物数量、食物数量、食物类型、投喂时间
- 刷新按钮

## 7. 绑定事件逻辑

### FeedPointApplyPanel.java（志愿者）
- 提交按钮 → 调用 FeedPointService.apply()

### FeedPointListPanel.java（管理员）
- 通过按钮 → 调用 FeedPointService.review(id, "已通过", opinion)
- 拒绝按钮 → 调用 FeedPointService.review(id, "已拒绝", opinion)
- 刷新按钮 → 重新加载投喂点列表

### FeedRecordInputPanel.java（志愿者）
- 提交按钮 → 调用 FeedRecordService.record()

### FeedRecordListPanel.java（志愿者）
- 刷新按钮 → 重新加载投喂记录列表

## 9. 正常流程测试

1. 志愿者登录后进入投喂点申请
2. 填写投喂点信息，点击提交
3. 投喂点申请成功
4. 管理员登录后进入投喂点管理
5. 查看投喂点申请列表
6. 点击"通过"按钮
7. 填写审核意见，点击确认
8. 投喂点审核通过
9. 志愿者登录后进入投喂记录录入
10. 选择投喂点，填写投喂信息，点击提交
11. 投喂记录录入成功
12. 查看投喂记录列表

## 10. 异常流程测试

1. 申请投喂点时地点为空 → 提示"投喂地点不能为空"
2. 申请投喂点时区域为空 → 提示"所在区域不能为空"
3. 申请投喂点时说明为空 → 提示"说明不能为空"
4. 录入投喂记录时宠物数量 < 1 → 提示"宠物数量必须 ≥ 1"
5. 录入投喂记录时食物数量 ≤ 0 → 提示"食物数量必须 > 0"
6. 录入投喂记录时投喂时间早于当前时间 → 提示"投喂时间不能早于当前时间"
7. 数据库操作失败 → 提示"操作失败，请重试"
