# 校园流浪猫狗投喂与领养管理系统 - 答辩演示脚本

## 1. 演示目标

通过最少的操作步骤，向老师完整展示系统的角色权限、主要业务流程、数据联动和实用价值，突出本项目“校园流浪动物投喂与领养闭环管理”的主题。

## 2. 演示账号准备

- 管理员：admin / admin123
- 普通用户：user01 / 123456
- 志愿者：vol01 / 123456

## 3. 推荐演示顺序

### 第一部分：项目介绍

可直接说明：

- 这是一个基于 Java Swing 开发的校园流浪猫狗投喂与领养管理系统
- 系统支持普通用户、志愿者、管理员三类角色
- 主要覆盖动物展示、领养申请、投喂管理、物资管理、公告通知、统计分析等功能

### 第二部分：管理员视角演示

演示内容：

1. 管理员登录
2. 查看并管理动物档案
3. 发布系统公告
4. 查看领养申请并进行审核
5. 查看投喂点申请或异常上报
6. 查看统计图表
7. 查看系统日志

讲解重点：

- 管理员负责整个系统的数据管理和审核管理
- 系统可以形成可追溯的管理闭环

### 第三部分：普通用户视角演示

演示内容：

1. 普通用户登录
2. 浏览动物列表
3. 查看动物详情
4. 提交领养申请
5. 查看个人申请状态
6. 查看审核消息和领养协议

讲解重点：

- 普通用户可以方便地查看校园流浪动物信息
- 用户可以直接发起领养流程并跟踪结果

### 第四部分：志愿者视角演示

演示内容：

1. 志愿者登录
2. 提交投喂点申请
3. 填写投喂记录
4. 上报异常情况
5. 查看处理结果
6. 查看物资库存或领取记录

讲解重点：

- 志愿者模块体现了校园日常救助与投喂管理过程
- 系统不仅支持领养，还支持长期救助管理

## 4. 建议重点展示的闭环

优先展示这一条主流程：

- 普通用户提交领养申请
- 管理员审核申请
- 普通用户查看审核结果与消息通知

这条流程最能体现系统完整性，也最适合答辩现场快速展示。

## 5. 老师可能会问的问题

### 问题一：为什么使用 Swing

回答建议：

- 因为本项目是 Java 期末作业，Swing 是 Java 自带的 GUI 技术，部署简单，适合作业演示

### 问题二：为什么用 MySQL

回答建议：

- 因为系统涉及多角色、多模块、多状态流转，使用 MySQL 更方便展示数据持久化与查询管理能力

### 问题三：系统亮点是什么

回答建议：

- 系统不只是动物信息展示，还完成了领养、投喂、异常处理、物资、公告和日志等全流程闭环

### 问题四：有哪些异常处理

回答建议：

- 做了用户名重复校验、手机号校验、重复申请校验、库存不足校验、删除确认、图片加载失败兜底等处理

## 6. 演示注意事项

- 提前导入测试数据
- 提前检查图片路径
- 提前检查数据库连接
- 尽量只展示最稳定的主流程
- 若时间不足，优先展示登录、动物浏览、领养申请、管理员审核

可以这样总结：

本系统围绕校园流浪猫狗的投喂和领养场景，完成了多角色、多模块的管理功能，实现了从信息展示、申请提交、后台审核到消息通知的业务闭环，满足 Java 课程项目对界面开发、数据库操作和完整业务流程的综合要求。
