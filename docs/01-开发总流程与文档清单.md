# 校园流浪猫狗投喂与领养管理系统 - 开发总流程与文档清单

## 1. 项目目标

本项目采用 Trae + AI 协同开发方式，按照需求文档完成一个基于 Java Swing 的桌面管理系统，支持普通用户、志愿者、管理员三类角色，覆盖动物展示、领养申请、投喂管理、物资管理、公告通知、统计报表、系统日志等完整流程。

## 2. 开发原则

- 先文档驱动，再编码落地
- 先搭主干架构，再做业务模块
- 先完成闭环流程，再补充扩展功能
- 先保证可运行，再优化界面和体验
- 所有模块都要能映射到需求文档中的功能点

## 3. 推荐开发流程

### 阶段一：需求整理与任务拆解

目标：把需求文档转成可执行任务。

产出：

- 开发任务拆解文档
- 系统设计说明书
- 数据库设计文档
- 界面设计文档
- AI 开发约束文档

### 阶段二：项目骨架搭建

目标：建立可持续迭代的工程结构。

工作内容：

- 初始化 Java 项目结构
- 建立 entity、dao、service、view、utils、db、config 等包
- 配置数据库连接与基础工具类
- 搭建登录页、主窗口、导航框架

### 阶段三：核心业务模块开发

目标：完成答辩必需的主业务闭环。

优先顺序：

1. 登录与注册
2. 动物档案展示与检索
3. 领养申请与管理员审核
4. 用户中心
5. 公告与消息通知
6. 投喂点申请与投喂记录
7. 异常上报处理
8. 物资管理
9. 后台数据管理
10. 图表统计与系统日志

### 阶段四：联调与测试

目标：验证流程闭环与异常场景。

工作内容：

- 校验角色权限
- 校验表单规则
- 校验状态流转
- 校验数据库读写
- 补充测试数据
- 修复界面与逻辑问题

### 阶段五：交付与答辩准备

目标：形成可演示、可提交的项目成果。

产出：

- 可运行源码
- 数据库脚本
- 使用说明
- 测试与验收文档
- 答辩演示脚本

## 4. 本项目建议里程碑

当前按照以下顺序推进：

1. 完成文档体系搭建
2. 输出数据库与界面设计
3. 开始初始化 Java 项目骨架
4. 优先开发登录、动物管理、领养流程
5. 再补投喂、物资、统计、日志等模块

## 7. 开发方式说明

本项目采用“文档驱动 + 模块迭代 + 每阶段验证”的方式推进：

- 每开发一个模块前，先确认该模块的数据表、页面、事件、校验规则
- 每开发一个模块后，立即验证增删改查与角色权限
- 所有功能尽量复用统一工具类与统一 UI 风格
- 全程以需求文档为基准，避免偏离期末作业目标
