# 消息通知模块开发文档

## 1. 页面与弹窗清单

### 消息中心面板
- **文件**：`src/main/java/com/campuspet/view/user/MessageCenterPanel.java`
- **功能**：展示用户的所有消息，支持查看、标记已读
- **布局**：表格展示 + 标记已读/全部已读按钮 + 刷新按钮

### 消息详情弹窗
- **文件**：`src/main/java/com/campuspet/view/user/MessageDetailDialog.java`
- **功能**：查看消息详细内容
- **布局**：标题 + 内容 + 发送时间 + 类型标签 + 标记已读按钮

## 2. 数据库表与字段

### message 表（已有）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户ID |
| title | VARCHAR(100) | 标题 |
| content | VARCHAR(500) | 内容 |
| type | VARCHAR(50) | 类型 |
| is_read | VARCHAR(10) | 是否已读 |
| send_time | DATETIME | 发送时间 |

## 6. 界面层开发

### MessageCenterPanel.java

**功能**：
- 表格展示所有消息
- 显示：标题、内容、类型、发送时间、是否已读
- 标记已读按钮 → 标记选中消息为已读
- 全部已读按钮 → 标记所有消息为已读
- 刷新按钮 → 重新加载消息列表

**布局**：
```
┌─────────────────────────────────────────┐
│  [标记已读] [全部已读] [刷新]            │
├─────────────────────────────────────────┤
│  ┌───────────────────────────────────┐  │
│  │ 标题 | 内容 | 类型 | 时间 | 已读 │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### MessageDetailDialog.java

**功能**：
- 显示消息标题
- 显示消息内容
- 显示发送时间
- 显示消息类型
- 标记已读按钮

## 7. 绑定事件逻辑

### MessageCenterPanel.java
- 标记已读按钮 → 调用 MessageService.markAsRead()
- 全部已读按钮 → 调用 MessageService.markAllAsRead()
- 列表项双击 → 打开 MessageDetailDialog
- 刷新按钮 → 重新加载消息列表

### MessageDetailDialog.java
- 标记已读按钮 → 调用 MessageService.markAsRead() + 关闭弹窗
- 关闭按钮 → 关闭弹窗

## 8. 表单校验

无需表单校验。

## 9. 正常流程测试

1. 用户登录后进入消息中心
2. 查看消息列表
3. 点击"标记已读"按钮
4. 选中消息标记为已读
5. 点击"全部已读"按钮
6. 所有消息标记为已读
7. 点击消息列表项
8. 消息详情弹窗打开
9. 点击"标记已读"按钮
10. 消息标记为已读，弹窗关闭

## 10. 异常流程测试

1. 标记不存在的消息 → 提示"消息不存在"
2. 数据库操作失败 → 提示"操作失败，请重试"

## 11. 消息类型

| 类型 | 说明 | 示例 |
|------|------|------|
| SYSTEM | 系统消息 | "系统升级通知" |
| ADOPT | 领养消息 | "您的领养申请已通过" |
| ANNOUNCEMENT | 公告消息 | "新公告：领养须知更新" |
| FEEDBACK | 回访消息 | "宠物回访提醒" |

## 12. 消息发送场景

1. **领养申请通过** → 发送 ADOPT 类型消息
2. **领养申请拒绝** → 发送 ADOPT 类型消息
3. **新公告发布** → 发送 ANNOUNCEMENT 类型消息
4. **宠物回访提醒** → 发送 FEEDBACK 类型消息
