# 校园流浪猫狗投喂与领养管理系统 - 开发任务拆解

## 1. 开发总目标

围绕需求文档中的十大模块，分阶段完成一个基于 Java Swing + MySQL + JDBC 的本地桌面系统，满足期末项目演示、运行和答辩要求。

## 2. 模块拆解总览

| 模块编号 | 模块名称 | 优先级 | 依赖模块 | 完成标准 |
| --- | --- | --- | --- | --- |
| M01 | 项目骨架与基础配置 | 高 | 无 | 工程可运行，数据库可连接，基础包结构建立完成 |
| M02 | 登录与注册 | 高 | M01 | 普通用户、志愿者可注册登录，管理员可固定账号登录 |
| M03 | 主界面与导航框架 | 高 | M02 | 登录后按角色进入对应主界面，左侧导航可切换 |
| M04 | 动物档案展示与检索 | 高 | M01、M03 | 动物列表、详情、搜索筛选、地图标记可用 |
| M05 | 领养申请与审核 | 高 | M02、M04 | 用户可申请，管理员可审核，结果可回写 |
| M06 | 用户中心 | 高 | M02、M05 | 用户能查看个人资料、申请记录、消息通知 |
| M07 | 公告与消息通知 | 中 | M03、M06 | 管理员可发公告，用户可查看消息 |
| M08 | 投喂点申请与投喂记录 | 中 | M02、M03 | 志愿者可申请投喂点并记录投喂日志 |
| M09 | 异常上报处理 | 中 | M08 | 志愿者可上报异常，管理员可处理 |
| M10 | 物资管理 | 中 | M03 | 物资入库、出库、库存预警、记录查询可用 |
| M11 | 后台数据管理 | 中 | M04、M05、M08、M10 | 管理员可统一管理动物、用户、申请、投喂点 |
| M12 | 图表统计与系统日志 | 低 | M05、M08、M10、M11 | 图表展示正常，关键操作有日志记录 |
| M13 | 测试数据与验收 | 高 | 全部 | 提供可演示数据和主流程验收结果 |

## 3. 分阶段任务

### 阶段 A：基础搭建

#### A1 工程结构初始化

- 建立 src 包结构
- 建立 resources 目录
- 建立 db.sql、config.properties 等配置资源
- 建立统一常量类、工具类、异常提示类

#### A2 数据库与连接层

- 建立数据库表结构
- 编写 JDBC 工具类
- 实现基础 DAO 通用方法
- 初始化管理员账号与测试数据

#### A3 基础 UI 框架

- 登录窗口
- 注册窗口
- 主窗口
- 左侧菜单栏
- 右侧内容切换面板

### 阶段 B：核心闭环

#### B1 登录注册模块

- 用户注册
- 用户名重复校验
- 手机号格式校验
- 管理员固定账号登录
- 按角色跳转不同界面

#### B2 动物展示模块

- 动物列表页
- 动物详情弹窗
- 条件筛选
- 关键词搜索
- 图片加载与默认占位图

#### B3 领养流程模块

- 领养申请弹窗
- 提交申请
- 防止重复申请
- 管理员审核列表
- 审核通过或拒绝
- 消息提醒
- 协议生成

#### B4 用户中心模块

- 我的信息
- 修改密码和手机号
- 我的领养申请
- 我的消息通知
- 领养回访查看

### 阶段 C：扩展业务

#### C1 投喂管理

- 投喂点申请
- 投喂点审核
- 投喂记录新增
- 投喂记录查询

#### C2 异常上报

- 异常上报表单
- 图片选择
- 管理员处理意见
- 处理结果通知

#### C3 物资管理

- 物资类型维护
- 物资入库
- 物资出库
- 库存表格展示
- 低库存预警
- 出入库记录查询

#### C4 公告管理

- 发布公告
- 编辑公告
- 删除公告
- 用户公告列表
- 新公告消息提醒

### 阶段 D：后台与统计

#### D1 后台统一管理

- 宠物信息管理
- 用户管理
- 投喂点管理
- 申请管理

#### D2 图表统计

- 宠物种类饼图
- 月度领养折线图
- 物资消耗柱状图
- 投喂点使用频率图
- 救助数量变化图

#### D3 系统日志

- 登录日志
- 提交申请日志
- 审核日志
- 物资出入库日志
- 查询筛选日志

## 4. 推荐开发顺序

建议按以下次序逐个实现：

1. 工程结构与数据库
2. 登录注册
3. 主界面框架
4. 动物档案管理与展示
5. 领养申请与审核
6. 用户中心
7. 公告与消息
8. 投喂管理
9. 异常上报
10. 物资管理
11. 后台统一管理
12. 图表统计与日志
13. 验收与打包

## 5. 每个模块统一执行模板

每个模块开发时固定按下面步骤推进：

1. 明确页面与弹窗
2. 明确数据库表与字段
3. 编写实体类
4. 编写 DAO
5. 编写 Service
6. 编写界面层
7. 绑定事件逻辑
8. 做表单校验
9. 测试正常流程
10. 测试异常流程
